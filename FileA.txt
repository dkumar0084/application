git init: Initialize a new Git repository in the current directory.
git clone <repository>: Create a local copy of a remote repository.
git add <file>: Add a file or changes to the staging area.
git commit -m "<message>": Commit the staged changes to the repository with a descriptive message.
git status: Show the current status of the repository, including any untracked or modified files.
git diff: Display the changes between the working directory and the staging area.
git diff --cached: Display the changes between the staging area and the last commit.
git log: Show the commit history of the repository.
git branch: List all branches in the repository.
git branch <branch-name>: Create a new branch.
git checkout <branch-name>: Switch to a different branch.
git merge <branch-name>: Merge changes from a specified branch into the current branch.
git remote add <remote-name> <remote-url>: Add a remote repository.
git push <remote-name> <branch-name>: Push the local commits to a remote repository.
git pull <remote-name> <branch-name>: Fetch and merge changes from a remote repository.
git fetch <remote-name>: Fetch changes from a remote repository without merging.
git reset <file>: Remove a file from the staging area, while keeping the changes.
git reset --hard: Discard all local changes and reset to the last commit.
git revert <commit>: Create a new commit that undoes the changes made in a previous commit.
git rm <file>: Remove a file from both the working directory and the repository.
These are just some of the commonly used Git commands. There are many more commands and options available. You can refer to the official Git documentation for a comprehensive list and more detailed explanations of each command: https://git-scm.com/doc
